Introduction

The Atter application is a solution intended to ease the organizing academic activities and it's attendees, providing a easy to use system for tracking attendances for both consumers, oraganizer and attendee.

The application is built in a way where all the data is provided in real-time making the attendance tracking more interactive. Also, we are able to provide statistics for attendance and possible frauds. The impelemtation for tracking an attendance uses the geolacation of the user device (with his approval), by doing so we are able to detect possible frauds. We feel that this approach is the best for an organizer that wants to have an accurate attendance statistics.

Architecture

Single-page application

Atter is made unsing its own Single-Page Appliation(SPA) built specifically to support its needs. We choose SPA over MPA(Multiple-Page Application) because SPA is fast, more interactive, can provie to the user a more pleasant experience, it's easier to develop a serverless application using SPA.

We are following a traditional way of implementing a SPA but we did some enhancements introducing built-in Dependency Injection system, component life-cycle hooks, style encapsulation for each component, a DOM renderer that makes sure that only the differences between what is already rendered and what should be rendered will affect the DOM improving performance and a template engine that help us to interpolate HTML with JS making the imporving development time and code readability and reliability.

Our SPA framework has the following components:

  • Application container
  • Router manager
  • Component mananger
  • Renderer
  • Renderable element equivalent to components from known SPA frameworks
  • Dependency Injection container
  • DOM processors, used to manipulate events like click, submit on an element from a renderable etc
  • Encapsulator, used to encapsulate style for renderables
  • Life-cycles hooks: AfterRender, OnInit, OnRefresh and OnDestroy
  • Template engine
